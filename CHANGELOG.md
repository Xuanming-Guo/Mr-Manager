# Changelog

All notable changes to Mr Manager will be documented in this file. The project follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and intends to use [Semantic Versioning](https://semver.org/) once releases begin.

## [Unreleased]

### Added

- Initial Tauri 2, Rust, React, and strict TypeScript foundation.
- Local-first architecture, privacy, safety, and phased delivery documentation.
- Frontend unit tests, a browser smoke test, and npm/Rust continuous integration checks.
- Repository contribution, conduct, and Apache-2.0 licensing files.
- Phase 7 Cleaner with bounded explicit-root scanning, deterministic candidates, immutable plans, verified quarantine, durable manifests, conflict-safe restore, confirmed purge, and fixture-only safety tests.
- Pre-Phase-9 usability and reliability pass with Rust-owned background tasks, automatic live-service topology, expanded process/port filters, partial Docker diagnostics, labelled network charts, Internet Test preflight, evidence explanations, and strongly confirmed whole-manifest Cleaner purge.

### Changed

- Renamed the product to Mr Manager while retaining legacy app-data identifiers and filenames so existing local data remains available.
- Renamed System Diagnostics, added a compatible `/war-room` redirect, and replaced its basic recording bars with a labelled responsive CPU/RAM timeline.
- Reduced refresh modes to Normal (2.5 seconds) and Fast (1 second), with a forward migration for legacy setting values.
- Removed introductory paragraphs beneath page titles and the Settings storage footnote.

### Removed

- Content Studio, README Forge, Portfolio Vault, and the unfinished AI, MCP, and Obsidian integration. Existing schema-5 records and application-data files remain untouched.

### Fixed

- Captured Windows CLI probes no longer create visible terminal windows during Docker, Network, Git, WSL, integration, GPU, or managed-process operations.

No public release has been made.
